Trusted Systems and Reference Monitor

What is a Trusted System?

  • A computer system that can be trusted to a specified extent to enforce a specified security policy

What is a Reference Monitor?

  • Entity at the heart of the system
  • Responsible for all decisions related to access controls

Suppose a subject would like to request for an action on an object, The reference monitor will check the request to determine if the subject has the privilege to do so. If he has, access is granted. If not, access is denied.

For this example, the owner requests for access to the admin side which the Reference Monitor checks if he/she is allowed access to it. Since the owner is allowed access to it, the Reference Monitors authorises his/her request.

On the other hand, the customer requested to enter the admin side is not authorised and thus, the Reference Monitors rejects the customer's request.

What are the 3 characteristics of a Reference Monitor?

  1. Should be tamperproof
  2. Should always be invoked
  3. Should be small enough so that it can be independently tested
